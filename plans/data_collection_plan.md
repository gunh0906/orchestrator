# 설비 데이터 수집 시스템 계획서

## 배경
- 머신오토앱과 모니터서버가 각각 LSV2 연결 → 충돌 (LSV2 1연결 제한)
- 해결: 서버가 LSV2 독점, 앱은 파일전송만 직접 + 설비정보는 서버 API
- 수집 데이터를 시계열 저장 → 추후 ML 활용 (공구마모 예측, 가공조건 최적화)

## 데이터 수집 구조

### 1. 자동 수집 - 서버 LSV2 (1~5초 주기)
| 항목 | 타입 | 단위 | 출처 | 비고 |
|------|------|------|------|------|
| 스핀들 RPM | float | rpm | LSV2 함수 | actual 평균 |
| 이송속도 | float | mm/min | LSV2 함수 | actual 평균 |
| 프로그램 번호 | string | - | LSV2 함수 | |
| 프로그램 줄 수 | int | lines | LSV2 함수 | 가공시간 예측용 |
| 공구 번호 | int | - | LSV2 함수 | |
| 공구 R값 | float | mm | LSV2 공구테이블 | |
| 공구 길이 | float | mm | LSV2 공구테이블 | |
| 공구 날수 | int | ea | LSV2 공구테이블 | FZ 계산용 |
| 프로그램 런타임 | float | sec | LSV2 함수 | 누적 |
| 스핀들 부하 | float | % | PLC 주소 (관리자 매핑) | 공구마모 핵심 |
| X축 부하 | float | % | PLC 주소 (관리자 매핑) | |
| Y축 부하 | float | % | PLC 주소 (관리자 매핑) | |
| Z축 부하 | float | % | PLC 주소 (관리자 매핑) | |
| 쿨런트 상태 | bool | on/off | PLC 주소 (관리자 매핑) | |
| 스핀들 모터온도 | float | °C | PLC 주소 (관리자 매핑) | |
| 스핀들 베어링온도 | float | °C | PLC 주소 (관리자 매핑) | |
| 설비 주변온도 | float | °C | PLC 주소 (관리자 매핑) | 예비 (추후) |

### 2. 자동 수집 - 머신오토앱
| 항목 | 타입 | 출처 | 비고 |
|------|------|------|------|
| 작업번호 | string | 폴더 지정 시 자동 | 데이터 묶음 키 |
| 파일전송 상태 | enum | 전송 시작/끝 | 서버에 알림 |

### 3. 수동 입력 - 머신오토앱 캠시트
| 항목 | 타입 | 단위 | 비고 |
|------|------|------|------|
| 소재 | string | - | SUS304, AL6061 등 |
| 가공 종류 | enum | - | 황삭/정삭/드릴/탭/헬리컬 등 |
| 피치 | float | mm | 나사/헬리컬 가공용 |
| 목표 VC | float | m/min | 의도값 (실제와 비교) |
| 목표 FZ | float | mm/tooth | 의도값 |
| 목표 표면조도 | float | Ra | 품질 예측용 |

### 4. 파생값 (ML 정제 시 계산, 수집 안 함)
- VC (절삭속도) = π × D × RPM / 1000
- FZ (날당이송) = Feed / (RPM × 날수)
- MRR (소재제거율)
- 가공시간 예측 = 줄수 기반 추정

## 데이터 연결 키
```
작업번호 + 설비ID + 타임스탬프 → 모든 데이터 연결
```
- 같은 작업번호로 며칠씩 사용 → 시작/종료 날짜 자동 기록
- 작업번호 변경 시 이전 작업 자동 완료 처리

## 관리자 설정
- 설비별 PLC 주소 매핑 UI (모니터서버 관리 화면)
- 설비 프로필 (같은 모델이면 복사 가능)
- 수집 주기 설정 (1초/5초/10초)
- 테스트 읽기 기능

## 파일전송 상태 표시
- 머신오토앱이 전송 시작/끝을 서버 API로 알림
- 대시보드에 "파일 전송 중" 뱃지 표시 (오프라인 대신)

## 구현 순서
1. DB 스키마 설계 (시계열 테이블, 설비 프로필, 작업번호, 캠시트)
2. PLC 주소 매핑 관리 API + UI
3. 서버 수집 엔진 (매핑 기반 PLC 읽기 + 시계열 저장)
4. 머신오토앱 → 서버 API 연동 (작업번호, 캠시트, 전송상태)
5. 대시보드 수집 현황 표시
6. ML 데이터 export 기능
